1. YJJ 工作交接

User

解释YJJ工作交接

Assistant

先在项目里找 “YJJ” 和 “工作交接” 相关文件，找到 YJJ工作交接 文件夹，里面包含交接文档、BDS 技术文档、ADM BWF 技术方案、空间音频算法规范、C++ 迁移规范、中间格式参考和计划图。

最终解释：YJJ 交接的是“空间音频算法 + 私有 BDS 容器 + 母带格式 + App 编辑原型”的技术底稿和部分 C++ 实现，后续重点是集成、验证、硬件联调和产品化。

  • VBAP 渲染与 C++ 算法：VBAP、limiter、模式切换平滑。
  • BDS 空间音频私有容器：多轨音频、3D 空间元数据、自动化轨道封装。
  • App 编辑模式初步方案：AURORA、Stage、Blueprint 三个方向。
  • ADM BWF 母带格式：标准 ADM + BW64/BWF，4 PCM 轨到 3 audioObject。

User

User

输入输出是啥

Assistant

HTML 输入是 BDS scene JSON，不是音频文件。它主要读取：

sample_rate
timebase
codec_configs
tracks
elements
elements[].automation
mix_presentations

输出是浏览器里的空间声场可视化、当前时间点每个对象的 azimuth/elevation/distance/gain、BDS 编码约束和自动化关键帧列表。它不会生成新文件。

Assistant

新增了“JUCE 内接 bds-codec 导出 .bds”层，并提供导出流程可视化。

新增核心文件：

  • cxw/src/cxw_bds_export_bridge.h
  • cxw/src/cxw_bds_export_bridge.cpp
  • cxw/web/bds_export_visualizer.html
  • cxw/data/bds_export_plan_demo.json
  • cxw/docs/bds_export_bridge.md

链路：

JUCE App
  输入：4 路 stereo interleaved float PCM
  输入：cxw 生成的 BDS scene JSON
    ↓
cxw::BdsExportBridge
    ↓
bds::scene_json::parse
bds::validateScene
bds::EncodeSession::setScene
bds::EncodeSession::setTrackPcm
bds::EncodeSession::finish
    ↓
输出：cxw_demo.bds

真实 .bds 导出需要：

cmake -S .\cxw\src -B .\cxw\build `
  -DCXW_HAS_BDS_CODEC=ON `
  -DCMAKE_PREFIX_PATH="<bds-codec-install>"

没有 bds-codec 时走 dry-run，生成 bds_export_dryrun/scene_json_for_bds_codec.json 和 bds_export_dryrun/bds_export_manifest.json。
